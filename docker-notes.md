## Докер, тут все своими словами 

**docker --version** - чекнуть версию  
**sudo docker build -t <имя> <путь>** - читает файл *(Dockerfile с параметрами запуска для докера который надо создать ранее)* и собирает из него готовый шаблон - образ с именем <name>  
**sudo docker run -d --name <имя контейнера> <имя образа из docker build -t>** - Берет образ <имя> который мы создали через `sudo docker build -t`  
**sudo docker ps** - посмотреть список всех работающих в данный момент контейнеров, их айди и скольки они уже запущенны   
**sudo docker logs <имя>** - выводит на экран все что код в приложении который я запустил в контейнере успело отобразить  
**sudo docker exec -it <имя> bash** - пробивает изоляцию и подключает терминал `-it` к командной оболчке `bash` внутри работающего контейнера
